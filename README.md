# Wireshark Network Investigation

## Project Overview

This project documents a network traffic investigation performed using Wireshark. The objective was to analyze packet captures, identify the most active hosts, examine network protocols, investigate DNS and HTTP communications, and determine whether any suspicious activity was present.

The investigation followed a structured methodology similar to the workflow used by Security Operations Center (SOC) analysts during an initial network traffic assessment.

---

## Objectives

- Analyze packet captures
- Identify the busiest hosts
- Review protocol hierarchy
- Investigate DNS traffic
- Analyze HTTP requests
- Identify external communications
- Document findings and observations

---

## Tools Used

- Wireshark
- Packet Capture (PCAP)
- GitHub

---

## Investigation Process

1. Reviewed packet overview
2. Identified active endpoints
3. Analyzed protocol hierarchy
4. Examined host conversations
5. Investigated DNS queries
6. Reviewed HTTP GET and POST requests
7. Inspected TCP Streams
8. Documented findings
